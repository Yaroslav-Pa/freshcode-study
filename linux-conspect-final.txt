CRUD

Create
    mkdir назва-1 назва-2 - створення папок / директорій (приймає шлях до папки)
    mkdir -p - створення проміжних папок за необхідності
    touch назва-1 назва-2 - "створення" файла
    nano шлях-до-файлу - чисто для текстових файлів
    команда-вивід > шлях - може створювати новий файл з інформацією з потоку виводу
    команда-вивід >> шлях - може створювати новий файл з інформацією з потоку виводу
Read
    ls
    nano шлях-до-файлу - чисто для текстових файлів
    cat шлях-до-файлу - показує один або кілька файлів в потоці виводу, якщо файлів декілька то воно їх конкатинує (склеює)
    less шлях-до-файлу - показує файл у новому термінальному вікні (виходити q або ctrl + c)
Update
    nano шлях-до-файлу - текстовий редактор в командному рядку
    команда-вивід > шлях - замінію дані з файлу на дані з потоку виводу
    команда-вивід >> шлях - додає до даних з файлу дані з потоку виводу
Delete
    rm шлях-до-файлу... - видаляє файли
    rm -r шлях-до-файлу... - видаляє файли або папки, також видаляє весь вміст папок (рекурсивне видалення)
    sudo rm -rf шлях-до-файлу... - безумовно видаляє файли і папки, користуватися з обрежністю


Конвеєрний оператор ( | )

 команда1 | команда2 - передає дані зі стандартного виводу команди1 на стандартний вхід команди2. Можна повторювати стільки скільки потрібно
 
 sort - команда для сортування тексту, приймає текст або з файлів (тоді треба вказати шлях до них ), або якщо файли не вказані то приймає дані зі стандартного входу
 
 grep ШАБЛОН - фільтрує текст з файлів або переданий зі стандатного входу лишаючи текст, у якому зустрічається ШАБЛОН
 
 history - команда повертає історію останніх декількох тисяч виконаних команд.
 
 env - виводить змінні оточення операційної системи
 
Глоббінг

    Спеціальні символи:
    
    ? - будь-який ОДИН символ
    * - будь-яка кількість будь-якийх символів
    [1234abc-x] - будь-який ОДИН символ з перелічених в дужках, можна вказувати діапазони
    [!34386] - будь-який ОДИН символ ОКРІМ перелічених в дужках, можна вказувати діапазони
    
    \ якщо поставити перед спец сиволом то він втратить свої особливі властивості (екранування)

Процеси

    Якщо після виконання команди не бачимо запрошення - то термінал заблокований процесом і ніяких інших дій виконувати не буде поки не звільниться.
    
    Ctrl + C - закриття процесу, який блокує термінал
    Ctrl + Z -  призупинення процесу, який блокує термінал
    
    jobs - показує завдання які були запущені в поточному терміналі
    
    fg [число] - повертає фокус термінала на певну задачу. Якщо на дали число то запускаєтсья остання задача. Блокує термінал
    bg [число] - відновлює процес на заднньому плані (у фоні). Дозволяє нормально працювати як з процесом так і з терміналом
    команда & - запуск команди на задньому плані

    xlogo
    xeyes
    
    ps - показує знімок з процесами
    
    ps axu - показує знімок з усіма процесами
    
    ps aux | grep назва_процесу - легкий спосіб отримати номер процесу
    
    kill [сигнал] номер_процесу... - посилає певний сигнал процесу(-ам). За замовчанням відправляє 15 сигнал
        Сигнали:
            15 - аналог хрестика в графічному інтерфейсі.Не працює для зависших процесів
            9 - моментальне закриття (вбивство) процеса. Працює для зависших процесів
            
    сигнали вводяться таким чином:
        -9
        -s 9
        --signal 9

    killall [сигнал] назва_процесу - посилає певний сигнал всім процесам з вказанним ім'ям. 
    
    Відкрийте і заморозьте один процес xeyes
    зупиніть його за допомогою команди kill
    відкрийте з заморозьте 2 процеси xlogo та вбийте їх одночасно
    
    
Сервіси

    systemctl status назва_сервіса - показує поточний статус певного демона або сервіса
              start назва_сервіса - запускає сервіс, для запуску необхідні права адміністратора
              systemctl stop назва_сервіса - зупияє сервіс, необхідні права адміністратора
              restart назва_сервіса - спочатку зупиняє а потім знову запскає сервіс, необхідні права адміністратора
              is-enabled назва_сервіса - чи знаходиться сервіс в автозапуску
              enable назва_сервіса - додає сервіс до автозапуску, необхідні права адміністратора
              disable назва_сервіса - забирає з автозапуску, необхідні права адміністратора
              
Права доступу

    ls -l - перегляд файлів з вказанням прав доступу
        
        Групи прав:
            1. власник файлу
            2. група власника файлу
            3. всі інші
            
        Кожна група прав скаладається з 3 основних прав:
            r - право на читання файлу
            w - право на запис  / зміну файлу
            x - право на виконання / запуск файлу
            
            відсутність певного права позначається -
            
            chmod набір_правил[,набір_правил] назва_файлу... - змінює права доступу для файлу. Змінювати може або власник файлу або root
            
            набір_правил записується так:
                1 символ:
                    u - власник
                    g - група
                    o - всі інші
                    a - всі одразу
                2 символ:
                    + - додати права
                    - - забрати права
                    = - перезаписати права на вказані
                подальші символи збігаються з правами, вказаними вище
                
            chmod a+rwx file2
            
            Створіть 3 файли і встановіть ім наступні права доступу:
            
            1 - власнику можна все, групі можна читати, іншім можна читати
            2 - власнику можна читати, групі можна змінювати і читати, іншим не можна нічого
            3 - власнику можна читати і виконувати, групі можна читати і виконувати, іншим можна виконувати
            
chown користувач:група назва_файлу... - змінює власника і групу файлу. Потребує прав доступу рівня рута            
            
sudo команда - виконати команду від імені рута

sudo su користувач - зміна користувача термінала
sudo su - зміна користувача на рута

Скріпти
    файли, яким надано право на виконання
    
    зазвичай щоб їх запустити ви маєте в терміналі вказати шлях до них, для відносних шляхів при цьому ./ обов'язкові.
    якшо перенести скрипти в одну з папок, вказаних в змінній оточення PATH то вони будуть працювати як звичайні лінуксовськи команди.
    

    sudo apt install назва_пакету - встановлю програму з репозиторію, або оновлює її
    
    sudo apt remove назва_пакету - видаляє пакет
    sudo apt purge назва_пакету - видаляє пакет з всіма його налаштуваннями
    sudo apt autoremove - видаляє непотрібні пакети
    
    apt list - показати список встановлених пакетів (ДУЖЕ великий)
    apt list --upgradable - показати список пакетів, які можна оновити
    
    sudo apt update - перевірити оновлення для встановлених пакетів, також використовується для оновлення репозиторіїв про які знає апт
    sudo apt upgrade - оновлює всі пакунки
